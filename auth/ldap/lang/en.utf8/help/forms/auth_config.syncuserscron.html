Sync users automatically via cron job

Enable this setting to activate a task in the cron which will automatically create and/or update user accounts based on records in the LDAP server. By default, this cron task will execute once daily at midnight (server time). Edit the record in the "auth_cron" table or use the optional command-line script supplied at htdocs/auth/ldap/cli/sync_users.php if you wish to schedule it to run at other times or with other settings.

This setting will have no effect if the cron is not running. See the installation guide for instructions on how to set it up.
