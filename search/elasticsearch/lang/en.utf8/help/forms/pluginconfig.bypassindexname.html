Bypass index name

Most of the time, this setting should be left blank.

If this setting is populated, then Mahara will load new index data into the Elasticsearch index of this name while still retrieving search results from the Elasticsearch index stored under the "Index name" field.

If you want to reindex the whole site in a new index, but don't want to bring your current search capability down until reindexing finishes, you can use this field to point the reindexing cron at a new index, while the search functionality continues to use the old index.

Once the queue is empty, copy this index's name to the $cfg->plugin_search_elasticsearch_indexname variable in your config.php.

This setting cannot be changed through the web interface. You may change it by setting a value in your config.php file for the following config variable:

$cfg->plugin_search_elasticsearch_bypassindexname.
